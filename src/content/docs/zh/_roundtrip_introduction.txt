**An Efficient and Insightful Guide to Responsive Images in 2023**

Check the sidebar for a deep dive. This is a quick reference for upgrading your `<img>` tags to handle various sizes and pixel densities of modern devices. You should [know that a pixel is not a pixel](/en/pixels-not-pixels) and that `devicePixelRatio` is more likely to be 3.875 than 1:1. Modern phones pretend to be 320-428 pixels wide (in CSS pixels) for legibility, but each CSS pixel represents many device pixels. 

> If your image size will never change no matter how narrow the browser window becomes, you should use [density descriptors](/en/density-descriptors). This is typically the case for logos, icons, and buttons.

Do you know your [breakpoints](/en/breakpoints)? You need these numbers to fine-tune the solutions below, at the very least the viewport width breakpoint, which is where the main content area stops growing.


### The Easy Wins

```html

<img src="img.jpg?w=480" 
        srcset="img.jpg?format=webp&amp;w=480 480w, 
                img.jpg?format=webp&amp;w=800 800w, 
                img.jpg?format=webp&amp;w=1200 1200w, 
                img.jpg?format=webp&amp;w=1600 1600w, 
                img.jpg?format=webp&amp;w=2000 2000w"

        sizes="(max-width: 700px) 100vw, 800px" alt="A dog riding a bike" />

<!-- We assume you have a breakpoint at 700px. 
     At this viewport width, this image will take up 100% of the width, but it is limited to 800 CSS pixels when the viewport is larger. -->

<!-- This assumes you're using an image server that complies with the RIAPI standards, such as Imageflow. Real-time image variant generation is key to developer mental health. -->
```
