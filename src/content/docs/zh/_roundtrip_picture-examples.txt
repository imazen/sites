Here are some examples and explanations of using the <picture> element: 

1. Example for responsive images: 

```html 
<picture>
  <source srcset="image-large.jpg" media="(min-width: 768px)" />
  <source srcset="image-medium.jpg" media="(min-width: 480px)" />
  <img src="image-small.jpg" alt="An image" />
</picture> 
```

Explanation: In this example, the <picture> element is used to provide different sources for images for different screen sizes. The <source> element specifies different sources with the "srcset" and "media" attributes to indicate which source should be used under what conditions. The <img> element is a fallback for older browsers or when the conditions specified by the <source> element are not met. 

2. Example for art direction: 

```html
<picture>
  <source srcset="image-wide.jpg" media="(orientation: landscape)" />
  <img src="image-portrait.jpg" alt="An image" />
</picture>
```

Explanation: In this example the <picture> element provides different sources for images based on the orientation of the device. The <source> element with the "media" attribute set to "(orientation: landscape)" specifies a source for wide screens, while the <img> element specifies a source for portrait screens. 

3. Example for different image formats: 

```html 
<picture>
  <source srcset="image.webp" type="image/webp" />
  <source srcset="image.png" type="image/png" />
  <img src="image.jpg" alt="An image" />
</picture>
```

Explanation: In this example, the <picture> element provides different sources for images in different formats. The <source> element with the "srcset" and "type" attributes specifies different image formats and sources. The <img> element is a fallback for older browsers that do not support the <picture> element. 

## Breakpoints 

In responsive design, breakpoints define how a website or application layout should change when the viewport size changes. Breakpoints are typically defined using media queries in CSS to apply different styles based on screen width. These breakpoints can be combined with the <picture> element to provide different sources for images for different screen sizes. 

For example, in the first example above, we use the "media" attribute to specify which source to use at which screen width. When the screen width is greater than or equal to 768 pixels, the "image-large.jpg" source is used; when the screen width is greater than or equal to 480 pixels but less than 768 pixels, the "image-medium.jpg" source is used; when the screen width is less than 480 pixels the "image-small.jpg" source is used. This allows us to provide appropriately sized images for each device, shorten page load times, and improve the user experience. 

Breakpoints can be defined at any screen size and multiple breakpoints can be used to create layouts that respond to a variety of devices. By combining breakpoints with the <picture> element, flexible and responsive designs can be created that look good on any device.
