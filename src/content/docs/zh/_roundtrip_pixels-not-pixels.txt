### Device Pixels ≠ Browser Pixels ≠ Image Pixels

Thanks to Apple, CSS and HTML pixels (`px`) **almost always map to multiple device/printer pixels**. My phone's devicePixelRatio is 3.875; [what's yours?](https://www.mydevice.io/)

On my desktop device, the devicePixelRatio changes with browser zoom, but defaults to 1.5 (after all, my `System > Display > Scale` is set to 150%).

Image files (excluding SVGs) are decoded into pixel grids. **Getting image pixels to display perfectly at a 1:1 ratio with device pixels is difficult, but using `srcset` can get you close to the desired effect**. If you have a real, compelling reason for that kind of perfect match, a bit of JavaScript can resize the HTML elements after loading, although this forbidden technique cannot be disclosed here.

> Other absolute units (in, pc, pt, mm, cm, Q) are defined as quantities relative to the CSS pixel and pretend that everything is 96dpi. Developers will avoid using absolute units, as it's better to use units derived from font size or viewport/container size. Mismatched units can break layouts if the user adjusts their `Accessibility > Text Size` multiplier, scales the browser, or rotates their device. Needless to say, you should do all of these things before publishing your site.
