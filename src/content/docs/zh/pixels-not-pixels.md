### 设备像素 ≠ 浏览器像素 ≠ 图像像素

多亏了 Apple 公司，CSS 和 HTML 像素（`px`）**几乎总是映射到多个设备 / 打印机像素上**。我的手机的 devicePixelRatio 是 3.875；[你的呢？](https://www.mydevice.io/)

我的桌面设备的 devicePixelRatio 在浏览器缩放时会改变，但默认为 1.5（毕竟我的 `系统 > 显示 > 缩放` 是 150%）。

图像文件（不包括 SVG）被解码为像素网格。**让图像像素以与设备像素的 1：1 比例完美显示非常困难，但使用 `srcset` 您可以接近想要的效果**，如果您有真正合理的理由需要那种完美匹配，一点 JavaScript 可以在加载后调整 html 元素的大小，虽然这种被禁止的技巧不能在此透露。

> 其他绝对单位（英寸、派卡、点、毫米、厘米、Q）是相对于 CSS 像素的数量定义的，并且假装一切都是 96dpi。开发者会避免使用绝对单位，因为最好使用从字体大小或视口 / 容器尺寸派生出的单位。不匹配的单位可能会破坏布局，如果用户调整其 `无障碍 > 文本大小` 倍数、缩放浏览器或旋转其设备。不用说，您应该在发布站点之前做所有这些事情。
