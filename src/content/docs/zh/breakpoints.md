### 背景

小屏幕和大屏幕需要不同的布局。在使用 `srcset` 和 `sizes` 时，我们需要知道布局何时发生变化。

Web 开发者必须决定在较小的屏幕上缩小、隐藏或重新定位什么内容 - 或者更常见的是，在较大的屏幕上扩展、揭示或添加什么内容。他们也只有很少的信息可供使用。用户是在平板电脑、手机横向模式还是小型浏览器窗口上？

我们必须选择一个布局发生变化的任意视口宽度。这称为 **断点**。它是 CSS 像素的任意数量（[不是设备像素](/zh/pixels-not-pixels)）。Polypane 有一篇[很好的文章](https://polypane.app/blog/the-breakpoints-we-tested-in-2021-and-the-ones-to-test-in-2022/#the-breakpoints-to-develop-on-in-2023)介绍常用的断点。

有些图像（比如标志、图标或按钮)可能对这些断点引起的布局变化免疫（适用于 [srcset 密度描述符](/zh/density-descriptors)）。

主要内容图像将受其容器的大小限制。通常，页面的主要内容区域在最大屏幕上将限制为某个宽度，即 `max-width`，但在小屏幕上，主要内容区域将填满整个视口。

如果在某些断点上有多列，计算有效尺寸规则将变得更加困难，因为图像占用的视口宽度的百分比将会改变。

### 简单方法

话虽如此，不要想得太多。您可能非常满意以下近似值：

1. 主列（或图像容器）在何时停止增长？在该视口宽度范围内，我们可以使用 `100vw` 作为图片 `sizes` 属性来表示图像占用整个视口宽度。
2. 容器达到的最大宽度是多少？我们可以将其设置为其他所有内容的固定 `width`。

如果您的答案是 1 为 700px，2 为 800px，您可以使用以下 `sizes` 属性：

```html
<!-- 这里使用 CSS 像素，不是设备像素或图像像素。-->
<img [...] sizes="(max-width: 700px) 100vw, 800px"  />
```



> 您可能认为浏览器可以根据 CSS 很好地为我们处理所有这些计算。不幸的是，浏览器在下载样式表之前就非常热衷于选择图像 URL。因此，我们必须自己进行计算，如果我们没有完美地做到这一点，那么它们也不要怪我们。
