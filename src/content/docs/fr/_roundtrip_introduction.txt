**An Efficient and Straightforward Guide to Adaptive Images in 2023**

Check the sidebar for a deeper dive. This is the quick reference for upgrading `<img>` tags to handle modern devices in all their variable sizes and pixel densities. You should [know that pixels != pixels](/en/pixels-not-pixels) and that `devicePixelRatio` is more likely to be around 3.875 than 1:1. Modern phones claim to be 320-428 pixels wide for readability (in CSS pixels), but they have many device pixels per CSS pixel.

> If your image never changes size, no matter the width you give the browser window, you should use a [density descriptor](/en/density-descriptors) instead. This is often suitable for logos, icons, and buttons.

Do you know your [breakpoints](/en/breakpoints)? You'll need those numbers to fine-tune the solution below - at minimum the viewport width breakpoint where the main content area stops growing.

### The easy method

```html

<img src="img.jpg?w=480" 
     srcset="img.jpg?format=webp&w=480 480w, 
             img.jpg?format=webp&w=800 800w, 
             img.jpg?format=webp&w=1200 1200w, 
             img.jpg?format=webp&w=1600 1600w, 
             img.jpg?format=webp&w=2000 2000w"

     sizes="(max-width: 700px) 100vw, 800px" alt="Dog Riding a Bike" />

<!-- We assumed you have a breakpoint at 700px. 
     And that below this viewport width, this image takes up 100% width, but
     when the viewport is wider, the image is capped at 800 CSS pixels. -->

<!-- This assumes you're using an RIAPI-compatible image server such as Imageflow.
     On-the-fly image variant production is essential for developer sanity. -->
```

{{Note that the term "RIAPI" is not commonly used in English and may be confusing to readers unfamiliar with the acronym. It may be better to rephrase as "This assumes you're using an image server that supports on-the-fly production of image variants, such as Imageflow."}}
