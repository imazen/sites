### Device Pixels ≠ Browser Pixels ≠ Image Pixels

Thanks to Apple, a CSS/HTML Pixel (`px`) **almost never equals a single device/printer pixel**. The devicePixelRatio of my phone is 3.875; [what's yours?](https://www.mydevice.io/).

The devicePixelRatio of my desktop changes when zooming in the browser, but is by default 1.5 (my `System > Display > Scale`, after all, is set to 150%).

Images (with the exception of SVG files) are decoded as a grid of pixels. **Unfortunately, it's difficult to display image pixels perfectly at a 1:1 ratio with device pixels - but with `srcset`, you can at least get *close*, and a bit of JavaScript can adjust the size of the HTML element after it loads if you have a good reason for that perfect implementation, although this forbidden art is not revealed here.**

> Other absolute [units (in, pc, pt, mm, cm, Q)](https://developer.mozilla.org/en-US/docs/Web/CSS/length) are defined on a certain number of CSS pixels and a screen resolution of 96dpi. Developers do not use absolute units because it's generally better to use units derived from font size or viewport/container dimensions. Mismatched units can destroy layout when the user adjusts their `Accessibility > Text Size` factor, the browser zooms, or the device rotates. Of course, you should do all of this before releasing your website.
