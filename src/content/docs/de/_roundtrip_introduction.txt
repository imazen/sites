**An Efficient and Presumptuous Guide to Images up to 2023**

Check the sidebar for detailed analysis. This is the quick reference to get these `<img>` tags up to modern devices with all their various sizes and pixel densities. You should [know that Pixel != Pixel](/en/pixels-not-pixels) and `devicePixelRatio` is more likely to be 3.875 than 1:1. Modern phones pretend to be CSS pixels of 320-428px wide, but have many device pixels per CSS pixel.

> If your image will never change its size no matter the width of the browser window, you should instead use a [density descriptor](/en/density-descriptors). This is often suitable for logos, icons, and buttons. 

Do you know what your [breakpoints](/en/breakpoints) are? You will need these numbers to match the solution listed below accurately, and at least the breakpoint at the width of the viewport where the main content stops growing.

### The Easy Method

```html
<img src="img.jpg?w=480"
     srcset="img.jpg?format=webp&w=480 480w,
             img.jpg?format=webp&w=800 800w,
             img.jpg?format=webp&w=1200 1200w,
             img.jpg?format=webp&w=1600 1600w,
             img.jpg?format=webp&w=2000 2000w"
     sizes="(max-width: 700px) 100vw, 800px"
     alt="Dog riding a bicycle" />
     
<!-- We presume you have a breakpoint at 700px.
     And that under this viewport width, this image takes up 100% width, but
     if the viewport is larger, the image is limited at 800 CSS pixels. -->

<!-- This assumes you use an RIAPI-compliant image server like Imageflow. Dynamic generation of image variants is essential for developer's sanity. -->
```
