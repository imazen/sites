**An efficient and controversial guide to images for 2023**

Check the sidebar for detailed analysis. This is a quick annotation to update `<img>` tags to handle modern devices with all their different sizes and pixel densities. You need to [know that pixels are not all equal](/en/not-all-pixels-are-equal) and that `devicePixelRatio` is more likely to be around 3.875 than 1:1. Modern phones pretend to be 320-428px wide for legibility (in CSS pixels) but have many device pixels for every CSS pixel.

> If your image never changes size, no matter how tightly you resize the browser window, you should use a [density descriptor](/en/density-descriptors) instead. This is often suitable for logos, icons, and buttons.

Do you know your [breakpoints](/en/breakpoints)? You'll need these numbers to fine-tune the solution below - and at a minimum the viewport width breakpoint at which the main content area stops growing.

### The easy way

```html
<img src="img.jpg?w=480"
     srcset="img.jpg?format=webp&amp;w=480 480w, 
             img.jpg?format=webp&amp;w=800 800w, 
             img.jpg?format=webp&amp;w=1200 1200w, 
             img.jpg?format=webp&amp;w=1600 1600w, 
             img.jpg?format=webp&amp;w=2000 2000w"
     sizes="(max-width: 700px) 100vw, 800px" 
     alt="Dog riding a bike">
<!-- We have assumed you have a breakpoint at 700px. And that below that viewport width this image takes up 100% of the width but when the viewport is wider, the image is capped at 800 CSS pixels -->
<!-- This assumes you're using an RIAPI-compatible image server like Imageflow. Generating image variants on the fly is essential for developer sanity. -->
```
