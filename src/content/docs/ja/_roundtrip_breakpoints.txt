### Background

Small screens require different layouts than large monitors. To achieve the purpose of `srcset` and `sizes`, you need to know the points at which the layout changes.

Web developers need to determine what to shrink, hide, or rearrange (or more generally, what to enlarge, show, or add) on smaller screens. They have very little information available to them. What about users on tablets or landscape phones, or with small browser windows?

You need to choose any viewport width at which the layout changes. This is called a **breakpoint**. It is an arbitrary number of CSS pixels (not [device pixels](/en/pixels-not-pixels)). Polypane has a fantastic article about frequently used breakpoints and [ones to develop on in 2023](https://polypane.app/blog/the-breakpoints-we-tested-in-2021-and-the-ones-to-test-in-2022/#the-breakpoints-to-develop-on-in-2023).

Some images, such as logos, icons, or buttons, may be exempt from layout changes caused by these breakpoints (and may be resolved through [density descriptors in srcset](/en/density-descriptors)).

Main content images are limited by the size of their container. Typically, a page's main content area is constrained by a `max-width` of a certain width on larger screens, while on smaller screens, the main content area takes up the whole viewport.

With multiple columns at several breakpoints, it becomes difficult to calculate the rules for valid sizes, as the percentage of viewport width occupied by the images changes.

### A simple approach

Don't overthink it. You'll likely be just fine with the following approximations:

1. At what size will the main column (or the image container) stop growing? Use `100vw` for the image's `sizes` attribute, so that the image takes up 100% of the viewport width, up to that viewport width.
2. What is the maximum width the container achieves? You can set that as a fixed `width` for everything.

If the answer to 1 is 700px and the answer to 2 is 800px, then you can use the following `sizes` attribute:

```html
<!-- These are CSS pixels, not device or image pixels! -->
<img [...] sizes="(max-width: 700px) 100vw, 800px"  />
```

> You might think that the browser could handle these calculations when processing stylesheets, but it's strongly keen on selecting the image URL **before** the stylesheet downloads. Therefore, we need to do the calculation ourselves and it won't be any good if it isn't perfect.
