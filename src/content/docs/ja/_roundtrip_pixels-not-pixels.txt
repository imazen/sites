### Device Pixels ≠ Browser Pixels ≠ Image Pixels

Thanks to Apple, CSS and HTML pixels (`px`) are mapped to **multiple device/printer pixels almost all the time**. My mobile phone's `devicePixelRatio` is 3.875. [Check yours here](https://www.mydevice.io/).

Desktop `devicePixelRatio`s vary according to browser zoom, but are usually 1.5x. (Mine is because my `System > Display > Scale` is set to 150%.)

Image files (excluding SVGs) are decoded onto a pixel grid. **Displaying image pixels perfectly at a 1:1 ratio with device pixels is very challenging, but can be achieved to some extent using `srcset`**. Alternatively, if you have good reason to do so, you can adjust the size of HTML elements with a little bit of JavaScript after loading. However, such taboo methods will not be disclosed here.

> Other absolute units (`in`, `pc`, `pt`, `mm`, `cm`, `Q`) are defined as considering the number of CSS pixels and 96dpi to be equal. Inconsistent units can break layouts if users adjust their Accessibility > Text Size settings, zoom the browser, or rotate the device. Needless to say, all of these should be tested before shipping the site.
