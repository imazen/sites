**An efficient and opinionated guide for images in 2023**

Check the sidebar tab for deep dives. This is a quick reference for upgrading your `<img>` tags to handle modern devices at all sorts of sizes and pixel densities. Understand that *pixel != pixel* and `devicePixelRatio` is likely more than 1:1. Modern phones pretend to be 320-428px wide for legibility (in CSS pixels), but they have multiple device pixels per CSS pixel. 

> If the size of the image doesn't change no matter how narrow the browser window gets, you may need to use [density descriptors](/en/density-descriptors). These are often suitable for logos, icons and buttons.

Do you know your [breakpoints](/en/breakpoints)? You’ll need those values to fine-tune the solutions below – at least the breakpoint at which growth in the main content area stops.

### The easy method

```html

<img src="img.jpg?w=480"
     srcset="img.jpg?format=webp&w=480 480w,
             img.jpg?format=webp&w=800 800w, 
             img.jpg?format=webp&w=1200 1200w, 
             img.jpg?format=webp&w=1600 1600w, 
             img.jpg?format=webp&w=2000 2000w"
     sizes="(max-width: 700px) 100vw, 800px" 
     alt="Dog riding a bike">

<!-- Assumes there is a 700px breakpoint and, if the viewport is smaller than this, 
     that the image should take up 100% of the width of the viewport but, 
     if the viewport is larger, the image is capped at 800 CSS pixels wide. -->

<!-- Assumes you are using a RIAPI compliant image server like Imageflow. 
     For sanity's sake, you'll want real time image variable generation. -->
```
